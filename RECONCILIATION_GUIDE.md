# Reconciliation & Cost Auditing Guide

This guide explains how to use the logs generated by `llm-proxy` to reconcile your usage with API provider bills and audit costs.

## 📊 Logging Overview

The proxy generates two types of logs in the `logs/` directory:

1.  **`calls.jsonl`**: Detailed machine-readable JSON records, including request and response bodies. **Requires `--log-payloads` to be enabled.**
2.  **`requests.log`**: A human-readable text summary for real-time monitoring, including tokens.

## 🔍 Key Identification Fields

In `logs/calls.jsonl`, each entry contains critical fields for reconciliation:

- `vendorRequestId`: The unique request ID returned by the API provider (e.g., `req_123...`). **Use this to match with provider dashboard records.**
- `vendorTraceId`: The internal trace ID from the provider (if available).
- `durationMs`: Total time taken for the request.
- `request.body.model`: The model used for the request.
- `response.body.usage`: Token usage statistics (if returned by the vendor).

## 🛠️ Automated Analysis

The project includes a utility script to identify which "Agent" or "Role" made the request by parsing the system prompt.

### Run Agent Analysis
```bash
node analyze-agent-logs.js
```

This script parses `logs/calls.jsonl` and groups requests by the agent name found in the `system` message.

## 📝 Manual Audit Workflow

1.  **Extract Vendor IDs**: Use `grep` or `jq` to extract all provider request IDs for a specific period.
    ```bash
    grep -o '"vendorRequestId":"[^"]*"' logs/calls.jsonl
    ```
2.  **Match with Provider Dashboard**: Log in to your provider's console (e.g., OpenAI, DeepSeek, etc.) and search for these IDs in the "Usage" or "Logs" section.
3.  **Verify Token Counts**: Compare the `usage` object in your local JSONL logs with the provider's bill to ensure total transparency.

## 💡 Tips for Better Auditing

- **Agent Tagging**: If you are developing with tools like OpenCode or OpenClaw, ensure they include a descriptive name in their system prompts. Our `analyze-agent-logs.js` tool is optimized to pick these up.
- **Log Rotation**: For high-volume environments, consider using a log rotation tool to manage the size of `logs/calls.jsonl`.
